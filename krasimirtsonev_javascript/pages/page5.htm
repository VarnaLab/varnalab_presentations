5. Application structure/workflow

  • входна точка на приложението
  • глобални функции (обекти)
  • използване на модули (полиморфизъм)

    • капсулиране на функционалност
    • ползване на API за достъп
    • намаляне до минимум зависимостта на един модул от друг
